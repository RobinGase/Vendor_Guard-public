SOC 2 TYPE II REPORT
Independent Service Auditor's Report
=====================================

Service Organization: CloudSafe Solutions B.V.
Headquarters: Amsterdam, The Netherlands
Report Period: 1 January 2025 – 30 June 2025
Prepared by: KPMG Netherlands N.V.
Report Date: 15 August 2025

--- SCOPE ---

This report covers CloudSafe Solutions B.V.'s cloud document storage and archival platform,
including all supporting infrastructure hosted in AWS EU-West-1 (Dublin, Ireland).

Trust Service Criteria covered: Security, Availability, Confidentiality

--- AUDITOR OPINION ---

In our opinion, the controls stated in the description were suitably designed throughout
the period 1 January 2025 to 30 June 2025, and operated effectively to meet the applicable
Trust Services Criteria.

--- COMPLEMENTARY USER ENTITY CONTROLS ---

Users of the system are responsible for:
- Managing their own user access and termination procedures
- Encrypting data prior to transmission where end-to-end encryption is required
- Reviewing CloudSafe-provided access logs on a monthly basis

--- SECTION 3: DESCRIPTION OF THE SYSTEM ---

Infrastructure
CloudSafe operates on Amazon Web Services (AWS) EU-West-1 region exclusively.
All production data is encrypted at rest using AES-256. Data in transit is protected
by TLS 1.3. No data is stored or processed outside the European Economic Area.

Access Controls
- Role-based access control (RBAC) implemented across all systems
- Multi-factor authentication (MFA) mandatory for all employees
- Privileged access management via CyberArk PAM solution
- Access reviews conducted quarterly; all access certified by line managers
- Separation of duties enforced between development, operations, and security teams

Change Management
- All changes follow a documented change management process
- Production changes require approval from two senior engineers
- Emergency changes require post-implementation review within 24 hours
- Code reviewed via pull request process; automated SAST scanning on all commits

Vulnerability Management
- External penetration test conducted annually (Fox-IT, September 2025)
- Automated vulnerability scanning via Tenable.io — weekly scans
- Critical CVEs remediated within 48 hours; High within 14 days
- No critical or high vulnerabilities were open for more than 14 days during the audit period

Incident Management
- Documented incident response policy; tested via tabletop exercise Q2 2025
- Security incidents classified by severity (P1/P2/P3)
- P1 (critical) incidents: client notification within 2 hours, internal escalation to CISO
- Security incidents reported to Dutch NCSC per regulatory obligations
- No P1 security incidents occurred during the audit period

Backup and Recovery
- Daily automated backups retained for 90 days, stored in AWS EU-West-2 (London)
- Backups encrypted with separate key from production
- Recovery tested quarterly; RTO of 4 hours and RPO of 1 hour validated in testing
- Full DR failover test conducted March 2025 — all systems restored within 3.5 hours

Sub-processors
- AWS (Amazon Web Services EMEA SARL) — infrastructure hosting, DPA signed
- Cloudflare Netherlands B.V. — CDN and DDoS protection, no personal data processed
- PagerDuty — on-call alerting, no personal data transmitted

--- SECTION 4: TESTS OF CONTROLS AND RESULTS ---

Control CC6.1 — Logical and Physical Access Controls
Test: Inspected access control matrix and reviewed 25 user accounts
Result: No exceptions noted. All accounts had appropriate access; no orphaned accounts found.

Control CC6.2 — New Access Provisioning
Test: Inspected 15 new-hire access provisioning tickets
Result: No exceptions noted. All provisioning approved by line manager before access granted.

Control CC6.3 — Access Removal
Test: Inspected 10 termination cases
Result: 1 exception noted: access for one terminated employee removed 3 days after termination
(SLA: same day). Remediated. Control considered effective overall.

Control CC7.1 — Vulnerability Management
Test: Reviewed vulnerability scan reports for audit period
Result: No exceptions noted. All critical and high vulnerabilities remediated within SLA.

Control CC7.2 — Incident Response
Test: Reviewed incident log and 5 incident tickets
Result: No exceptions noted. All incidents classified and escalated per policy.

Control A1.1 — Availability — Capacity Management
Test: Reviewed capacity reports and auto-scaling configuration
Result: No exceptions noted. System availability was 99.97% during audit period.
